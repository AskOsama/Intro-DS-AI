When Goals Conflict

Sometimes you can't optimize everything at once!

Card Game Example

Goal 1: Be FAST (flip quickly, don't think too long)

Goal 2: Be ACCURATE (remember everything, make perfect matches)

CONFLICT:

  • Thinking longer = Better memory = More accurate
  • Thinking longer = Slower overall time

You can't maximize BOTH speed AND accuracy!

Real-World Example: Buying a Car ğŸš—

Goal 1: MAXIMIZE Speed (want fast car)
Goal 2: MINIMIZE Cost (want cheap car)

The Trade-off:
- Want MORE speed? Pay MORE money!
- Want LESS cost? Get LESS speed!

Pareto Front (Ø¬Ø¨Ù‡Ø© Ø¨Ø§Ø±ÙŠØªÙˆ)

The Pareto Front = All solutions where you can't improve one goal without hurting another

For Dominated Solutions: There exists another solution that is better in ALL objectives.

For Pareto Optimal Solutions: Any improvement in one objective requires sacrifice in another.

9. Why This Matters for AI

9.1 Machine Learning Training

Training a Neural Network = SEARCH PROBLEM!

  • Search Space: All possible weight values
  • Constraint: Must fit the training data
  • Objective: Minimize prediction error
  • Heuristic: Gradient descent (follow the slope down)

9.2 Reinforcement Learning

Robot Learning to Walk = SEARCH PROBLEM!

  • Search Space: All possible movements
  • Constraint: Don't fall over, stay within bounds
  • Objective: Maximize distance traveled
  • Heuristic: Actions that kept balance before â†’ try again!

Similar to card game: Robot "remembers" what worked, uses that knowledge to make better decisions, balances exploration (try new things) and exploitation (use what works)

11. Practice Questions

Q1: In an 8-card game, you flip card 1 (shows 5) and card 2 (shows 8). No match. Next, you flip card 3 (shows 5). What should a memory heuristic tell you to do?

Answer: Flip card 1! You remember card 1 was also a 5. This is a guaranteed match!

Q2: Why is random search sometimes useful even though it seems "dumb"?

Answer: Random search can help escape local traps and explore parts of the search space that systematic search might miss. It adds flexibility and can find unexpected solutions.

Q3: You're optimizing for both SPEED and ACCURACY. Can you achieve 100% in both?

Answer: Usually no! These goals often conflict. Going faster means less time to think, which reduces accuracy. This is why we have Pareto fronts - to show the best possible trade-offs.
