القسم 1

أطر التحسين متعدد الأهداف بـ Python

تقدم Python نظاماً بيئياً غنياً من أطر التحسين للتحسين متعدد الأهداف والخوارزميات الفوقية. تتراوح هذه الأدوات من المكتبات البحثية إلى الحلول الجاهزة للإنتاج.

jMetalPy

MIT نشط

التركيز: التحسين متعدد الأهداف باستخدام الخوارزميات الفوقية

  • دعم التحسين الديناميكي
  • الحوسبة المتوازية (Apache Spark، Dask)
  • التصور الفوري والتفاعلي
  • المعالجة الإحصائية اللاحقة
  • التعبير عن التفضيلات لصنع القرار

الخوارزميات: NSGA-II, NSGA-III, SMPSO, OMOPSO, MOEA/D, SMS-EMOA, GDE3, SPEA2, HYPE, IBEA

الموقع: github.com/jMetal/jMetalPy

DEAP (خوارزميات تطورية موزعة في Python)

LGPL-3.0 نشط

التركيز: الحوسبة التطورية العامة (ليست متمحورة حول التعدد)

  • التوازي وتوزيع الحمل
  • إطار عمل معياري وقابل للتوسيع
  • النماذج الأولية السريعة

الخوارزميات: GA, GP, CMA-ES, NSGA-II, SPEA2, MO-CMA-ES

الموقع: github.com/DEAP/deap

PyGMO / Pagmo

GPL-3.0 نشط

التركيز: التحسين الفردي/متعدد الأهداف مع المعالجة المتوازية

  • التحسين المتوازي بشكل ضخم
  • الأساليب الإحصائية لتنافس الخوارزميات
  • مؤشرات الجودة وتحليل مشهد اللياقة
  • متغيرات الخوارزميات الهجينة

الخوارزميات: GA, DE, PSO, SA, ABC, IHS, MC, CMA-ES, NSGA-II, MOEA/D

الموقع: esa.github.io/pagmo2/

Platypus

GPL-3.0 نشط

التركيز: التحسين متعدد الأهداف

  • المعالجة المتوازية في تقييم الحلول
  • التصور والإحصائيات
  • طرق تعتمد على نقطة مرجعية

الخوارزميات: CMA-ES, NSGA-II, NSGA-III, GDE3, IBEA, MOEA/D, OMOPSO, EpsMOEA, SPEA2

الموقع: github.com/Project-Platypus/Platypus

Pymoo

Apache 2.0 نشط

التركيز: التحسين متعدد الأهداف مع التعبير عن التفضيلات

  • طرق النقطة المرجعية
  • التصور والإحصائيات
  • معلومات التدرج عبر التفاضل التلقائي
  • اتخاذ القرار متعدد المعايير

الخوارزميات: GA, DE, NSGA-II, NSGA-III, U-NSGA-III, R-NSGA-III

الموقع: pymoo.org

Inspyred

MIT غير نشط

التركيز: الذكاء الحسابي المستوحى من الأحياء (غير متخصص في التعدد)

  • التركيز التعليمي والبحثي
  • واجهة برمجة بسيطة للبدء

الخوارزميات: GA, ES, PSO, ACO, SA, PAES, NSGA-II

الموقع: github.com/aarongarrett/inspyred

Geatpy

MIT نشط

التركيز: الخوارزميات الجينية والتطورية

  • أداء عالٍ مع خلفية Numpy
  • دعم التحسين متعدد الأهداف

الخوارزميات: GA, MOEA

الموقع: geatpy.com

القسم 2

إضافات ومحللات Excel

يوفر Microsoft Excel قدرات تحسين مدمجة يمكن توسيعها بإضافات الطرف الثالث للمسائل الأكثر تعقيداً.

الخيارات المجانية / مفتوحة المصدر

محلل Excel المدمج

مجاني (مضمّن)

المطور: Frontline Systems (لـ Microsoft)

  • البرمجة الخطية (Simplex LP)
  • التحسين غير الخطي (GRG Nonlinear)
  • الخوارزميات التطورية

القيود: 200 متغير قرار، 100 قيد

الأفضل لـ: التعلم، مسائل التحسين البسيطة

OpenSolver

GPL مجاني

المطور: جامعة أوكلاند

  • يستخدم محرك التحسين COIN-OR CBC
  • لا قيود مصطنعة على حجم المسألة
  • متوافق مع نماذج محلل Excel الموجودة
  • مُصوّر نموذج مدمج
  • وضع QuickSolve لإعادة الحل بشكل أسرع

المحللات المدعومة: CBC, Gurobi, CPLEX, XPRESS, GLPK, Bonmin, Couenne

الموقع: opensolver.org

SolverStudio

مجاني

المطور: جامعة أوكلاند

  • يدعم لغات نمذجة متعددة (PuLP, AMPL, GAMS, GMPL, Pyomo, Gurobi Python)
  • الحل السحابي عبر خوادم NEOS
  • النموذج مخزن داخل مصنف Excel
  • أنسب للمسائل الكبيرة والمعقدة
الموقع: solverstudio.org

أدوات التحسين في Google Sheets

تقدم Google Sheets العديد من إضافات التحسين التي تمكّن الحل السحابي للبيئات التعاونية.

OpenSolver لـ Google Sheets

مجاني

المطور: جامعة أوكلاند

  • Glop - محلل البرمجة الخطية من Google
  • SCIP - البرمجة الصحيحة المختلطة
  • COIN-OR Cbc - عبر سحابة NEOS
  • GLPK.js - يعمل محلياً في المتصفح

القيود: مهلة 6 دقائق للسكربت (مع إمكانية الاستئناف)

التثبيت: Google Workspace Marketplace

إضافة Solver (Frontline Systems)

مجاني جزئياً

المطور: Frontline Systems

  • محلل Simplex LP (خطي/صحيح مختلط)

  • طريقة البداية المتعددة (التحسين الشامل)
  • المحلل التطوري

القيود: 200 متغير، 100 قيد، 30 ثانية وقت الحل

التثبيت: Google Workspace Marketplace

القسم 4

لغات النمذجة

توفر لغات النمذجة الجبرية طريقة عالية المستوى لوصف مسائل التحسين التي يمكن حلها بواسطة محللات مختلفة.

اللغةالنوعالوصفالميزات الرئيسية
AMPL تجاري لغة النمذجة القياسية الصناعية عرض تجريبي مجاني متاح، دعم واسع للمحللات
GAMS تجاري نظام النمذجة الجبرية العام قوي للنماذج المعقدة، توثيق شامل
PuLP مفتوح المصدر نمذجة LP مبنية على Python مضمن مع SolverStudio، مناسب للمبتدئين
GMPL مفتوح المصدر لغة GNU MathProg (مجموعة فرعية من AMPL) مجاني، يعمل مع محلل GLPK
Pyomo / COOPR مفتوح المصدر نمذجة التحسين المبنية على Python نماذج مجردة، البرمجة العشوائية، دعم DAE
Gurobi Python تجاري واجهة Python الأصلية لـ Gurobi تكامل وثيق مع محلل Gurobi، أداء عالٍ
نصيحة: للتعلم، ابدأ بـ PuLP أو GMPL (كلاهما مجاني). للإنتاج، فكر في AMPL أو Pyomo حسب تفضيلك للغة متخصصة في المجال مقابل Python.
القسم 5

خدمات التحسين السحابية

توفر الخدمات السحابية الوصول إلى محللات قوية دون تثبيت محلي، مما يمكّن التعاون وقابلية التوسع.

NEOS Server

مجاني

خدمة مجانية عبر الإنترنت لحل مسائل التحسين.

  • أكثر من 60 محلل لأنواع مختلفة من المسائل
  • الوصول عبر الويب أو البريد الإلكتروني أو API
  • يدعم صيغ AMPL و GAMS وغيرها
  • يتكامل مع SolverStudio و OpenSolver
الموقع: neos-server.org

Frontline Solver Server

تجاري

التحسين المستند إلى الخادم من Frontline Systems.

  • نقل الحل إلى خادم بعيد
  • تنزيل النتائج تلقائياً
  • ميزات للمؤسسات

Satalia SolveEngine

تجاري

منصة تحسين سحابية.

  • متاح عبر OpenSolver لـ Google Sheets
  • قدرات تحسين للمؤسسات
  • وصول API

ملخص الخدمات السحابية

الخدمةالتكلفةالمحللاتالوصول
NEOS Serverمجاني60+ويب، API، SolverStudio
Frontline Serverتجاريمحركات Frontlineمنتجات Frontline
Satalia SolveEngineتجاريمتنوعةAPI، Google Sheets

القسم 7

التوصيات حسب حالة الاستخدام

للتعلم والتعليم

  • Excel: محلل Excel المدمج أو OpenSolver
  • Python: Pymoo أو DEAP
  • Google Sheets: إضافة Frontline Solver

للبحث والنماذج الأولية

  • التحسين متعدد الأهداف: jMetalPy, Pymoo, Platypus
  • الخوارزميات التطورية العامة: DEAP, PyGMO
  • Excel مع لغات النمذجة: SolverStudio

للإنتاج / المؤسسات

  • Excel: Premium Solver Platform + Gurobi/CPLEX/XPRESS engines
  • Python: PyGMO (متوازي)، Gurobi Python API
  • مستقل: AMPL, GAMS مع محللات تجارية
